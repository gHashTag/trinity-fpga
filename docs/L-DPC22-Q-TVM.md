## 2. MAX-TRUE Fabric Shape

The target is declared in `tools/tvm_max_true.py`:

```
FABRIC_NAME      = "MAX-TRUE"
TOTAL_CELLS      = 32          # GF16 ALU cells
QUAD_MESHES      = 2           # 2 × quad_mesh
BANKS            = 4           # 4 SRAM banks
TILES_PER_BANK   = 4           # 4 pipeline tiles per bank
GF16_BPW         = 4           # 4 bits per weight (GF16)
CLOCK_MHZ        = 50          # 50 MHz
ENERGY_NJ_PER_OP = 1.0         # 1 nJ / GF16 MAC
PEAK_GOPS        = 1.6         # 32 cells × 50 MHz = 1.6 GOPS
```

Cell addressing: `cell_id = quad × 16 + bank × 4 + tile`.
The full round-trip `cell_id ↔ (quad, bank, tile)` is tested in `_self_test()`.

## 3. BitNet Stub

Three layers with ternary weights `{-1, 0, +1}` from seed `0xBEEF`:

| Layer | in_dim | out_dim | lane | SRAM src | SRAM dst |
|-------|--------|---------|------|----------|----------|
| embed | 4 | 4 | 0 | 0x000 | 0x100 |
| attn  | 4 | 4 | 1 | 0x100 | 0x200 |
| ffn   | 4 | 4 | 2 | 0x200 | 0x300 |

With `in_dim = 4` (= GF16 `dot4` width), each output neuron produces exactly
4 ops: `load → dot4 → scale → store`.
Total: 4 neurons × 4 ops × 3 layers = **48 ops**, 4 unique opcodes.

---

## 4. Op Codes

| Opcode | Description | Latency (cycles) |
|--------|-------------|-----------------|
| `load` | SRAM → cell register | 2 |
| `dot4` | 4-element GF16 dot product (acc += Σ aᵢ·wᵢ) | 1 |
| `scale` | acc × float scale factor (dequantisation) | 2 |
| `store` | acc → SRAM | 2 |
| `add` | scalar add to acc (declared, not used by this stub) | 1 |
