## Breaking Changes

### CLI Reorganization

- **Old:** `trinity-cli` command
- **New:** `tri` unified command
- **Impact:** All command invocations must use `tri` instead of `trinity-cli`

### VIBEE Spec Format

- Added `sacred_math` section (optional)
- New `behaviors` format with `given/when/then`
- Old format still supported but deprecated

**Migration:**

```yaml
# Old format (deprecated)
behaviors:
  - my_function: "description"

# New format (recommended)
behaviors:
  - name: my_function
    given: Precondition
    when: Action
    then: Expected result
```

### API Changes

- **C API:** `trinity.h` → `trinity_vsa.h`
- **Function signatures:** Updated for HybridBigInt
- **Type names:** Unified naming convention

**Migration:**

```c
// Old (deprecated)
#include <trinity.h>
TrinityVector* vec = trinity_vector_create(10000);

// New (recommended)
#include <trinity_vsa.h>
TrinityHypervector* vec = trinity_hypervector_create(10000);
```

### Configuration Migration

- **Old:** `~/.trinity/config.json`
- **New:** `~/.trinity/config.v1.toml`

**Auto-migration:**

```bash
tri migrate-config
```

### Memory Layout

- HybridBigInt changes internal representation
- Old code using raw trit arrays needs update
- Use SDK API for compatibility
