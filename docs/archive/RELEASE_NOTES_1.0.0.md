## Installation

### Quick Install (Recommended)

#### macOS (Homebrew)

```bash
brew tap gHashTag/trinity
brew install tri
```

#### Arch Linux (AUR)

```bash
yay -S tri-cli
```

#### npm (Cross-Platform)

```bash
npm install -g @trinity-cli/tri
```

### Build from Source

```bash
# Prerequisites: Zig 0.15.x
git clone https://github.com/gHashTag/trinity.git
cd trinity
zig build                    # Build all targets
zig build tri                # Build TRI CLI
zig build test               # Run all tests
zig build bench              # Run benchmarks
```

### Docker

```bash
docker pull ghcr.io/ghashtag/trinity-node:latest
docker run -d --name trinity-node \
  -p 8080:8080 -p 9090:9090 -p 9333:9333/udp -p 9334:9334 \
  -v ~/.trinity:/data \
  ghcr.io/ghashtag/trinity-node:latest
```

See [INSTALLATION_1.0.0.md](./INSTALLATION_1.0.0.md) for detailed installation instructions.

## Migration Guide

### From v0.x to v1.0.0

#### Breaking Changes

1. **CLI Reorganization**
   - Old: `trinity-cli` → New: `tri`
   - All commands now unified under single binary

2. **VIBEE Spec Format**
   - Added `sacred_math` section
   - New `behaviors` format with `given/when/then`

3. **API Changes**
   - C API renamed: `trinity.h` → `trinity_vsa.h`
   - Function signatures updated for HybridBigInt

#### Migration Steps

```bash
# 1. Update dependencies
brew update && brew upgrade tri
# OR
yay -S tri-cli

# 2. Update VIBEE specs
# Add sacred_math section to existing specs

# 3. Update C API includes
# Old: #include <trinity.h>
# New: #include <trinity_vsa.h>

# 4. Rebuild custom extensions
zig build
```

#### Configuration Migration

```bash
# Old config location
~/.trinity/config.json

# New config location
~/.trinity/config.v1.toml

# Auto-migration
tri migrate-config
```

## Known Limitations

### Current Limitations

1. **Windows Support**
   - Native Windows support in development
   - Use WSL2 for production (fully supported)

2. **GPU Acceleration**
   - Current release is CPU-only
   - GPU support planned for v1.1.0

3. **Model Size**
   - Optimal for models up to 70B parameters
   - Larger models require distributed cluster

4. **Language Support**
   - Core: English (100%)
   - Partial: Russian, Chinese (90%+)
   - Other languages via automatic detection

### Performance Notes

1. **First Run**
   - JIT compilation adds ~100ms overhead
   - Subsequent calls use cached compilation

2. **Memory Requirements**
   - Minimum: 4GB RAM
   - Recommended: 16GB+ RAM
   - Large models: 32GB+ RAM

3. **Threading**
   - Maximum threads: min(CPU cores, 13)
   - Based on φ × 8 = 13 sacred limit
