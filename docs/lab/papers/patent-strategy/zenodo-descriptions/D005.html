Problem Statement

Vector Symbolic Architectures (VSA) enable compositional distributed representations but existing implementations use binary {0, 1} vectors (Kanerva 2009) or real-valued vectors (MAP/Gayler 2003, HRR/Plate 1995). Binary VSA wastes representational capacity (1 bit/element vs theoretical 1.58 bits for balanced ternary). Real-valued VSA requires floating-point arithmetic and 32 bits per element, making hardware acceleration expensive. No prior VSA implementation combines balanced ternary {-1, 0, +1} with SIMD acceleration.

Technical Disclosure

Core Operations

All operations are defined over vectors in {-1, 0, +1}d where d is the vector dimension:

  • Bind(a, b): Component-wise ternary multiplication. a[i] × b[i] using ternary multiplication table. Self-inverse: unbind(bind(a,b), b) = a. Used to create associations.
  • Unbind(bound, key): Identical to bind (self-inverse property). Retrieves associated vector from a binding.
  • Bundle(a, b, ...): Component-wise majority vote. For 2 vectors: ties broken by random ternary value. For 3+ vectors: strict majority. Creates superposition (set union analog).
  • Permute(v, count): Cyclic left rotation by count positions. Used for sequence encoding (position-dependent binding).
  • Similarity(a, b): Cosine similarity computed over ternary vectors, range [-1, 1].

SIMD Acceleration

Ternary elements are packed 2 bits each. SIMD processes 32 trits per iteration using 64-bit registers:

  • Encoding: +1 → 01, 0 → 00, −1 → 11 (2-bit balanced)
  • Bind: bit-parallel ternary multiplication via XOR + AND masks
  • Bundle: parallel majority vote with carry-save arithmetic
  • 32 trits/iteration throughput on 64-bit SIMD

Obvious Extensions (Picket Fence)

  • GPU acceleration: CUDA/ROCm kernels for ternary VSA operations with shared memory tiling
  • Higher-order balanced systems: base-5 {-2,-1,0,+1,+2} or base-7 VSA with 3-bit encoding
  • Streaming VSA: incremental bind/bundle over data streams without materializing full vectors
  • Hierarchical VSA: multi-resolution binding with different dimension sizes at each level
  • VSA-based attention: replacing Q·KT dot product with VSA similarity in transformers
  • Sparse ternary VSA: exploiting zero-density for further compression (CSR/bitmap format)
  • ARM NEON / RISC-V V-extension SIMD variants
