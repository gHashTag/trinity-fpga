Problem Statement

Population-Based Training (PBT, Jaderberg 2017 / US12314856B2) mutates hyperparameters (learning rate, batch size) but keeps the training objective fixed. ASHA (Li 2020) performs successive halving to prune underperforming trials but does not mutate configurations. Neural Architecture Search (NAS, Zoph 2017) searches model architecture but not the training objective. No existing system mutates the training objective itself (NTP, JEPA, NCA, hybrid) within a population of workers while simultaneously applying diversity-preserving injection and sacred 3^k constraints on the search space.

Technical Disclosure

Claim 1: Objective Mutation in PBT Population

Each worker in the population carries a config.objective field accepting values from {NTP, NTP-81+, JEPA, hybrid}. During injection (batch creation of new workers), the objective is selected according to DEFAULT_QUOTAS: 40% NTP, 20% NTP-81+, 20% JEPA, 20% hybrid. Mutation via mutateConfigSacred() can change the objective type, not just scalar hyperparameters. This enables evolutionary search over the training objective space — a dimension not explored by any prior PBT system.

Claim 2: Diversity-Preserving Injection with PPL=0.0 Sentinel

After recycling (killing underperformers), new workers are injected with PPL=0.0 sentinel value (line 4649 of source). This prevents freshly injected workers from being immediately re-killed as "worst" in the next evolution cycle. Diversity health metric H = product(stdev_lr, stdev_gc, stdev_wu) measures population spread across learning rate, gradient clipping, and warmup dimensions. When H falls below threshold, auto-injection triggers to restore diversity.

Claim 3: Sacred 3^k Constraints on Search Space

The evolutionary search space is constrained to sacred values derived from powers of 3:

  • SACRED_RUNGS: evaluation checkpoints at 3^k steps: 2187 (3^7), 6561 (3^8), 19683 (3^9), 59049 (3^10)
  • phi-grid LR: learning rates on a golden-ratio grid: base * phi^p, p in {-3, -2, -1, 0, 1, 2, 3}
  • Sacred warmups: {243 (3^5), 729 (3^6), 2187 (3^7)}
  • Sacred contexts: {27 (3^3), 81 (3^4), 243 (3^5)}

These constraints reduce the search space to values empirically shown to produce resonant performance in ternary networks (see D001: Ternary Resonance Law).

Claim 4: ASHA + PBT + Objective Injection Combined

SEVO combines three mechanisms in a single evolutionary loop:

  • ASHA: DEFAULT_RUNGS define successive halving checkpoints for early stopping of poor trials
  • PBT: mutateConfigSacred() perturbs hyperparameters of surviving workers, copying weights from top performers
  • Objective injection: computeQuotaDeficit() identifies underrepresented objective types and injects new workers to maintain diversity quotas

No prior system combines all three: ASHA prunes, PBT mutates scalars, SEVO additionally mutates the objective and enforces diversity quotas.

Claim 5: HSLM_FRESH Detection (Objective/Context Change)

Automatic detection of whether a mutated worker requires fresh training or can resume from checkpoint:

  • obj_changed || ctx_changed → FRESH=1 (restart from scratch — incompatible checkpoint)
  • Scalar-only change (LR, GC, warmup) → FRESH=0 (resume from existing checkpoint)

This minimizes wasted compute: objective changes require retraining, but hyperparameter changes can warm-start from the parent checkpoint.

Obvious Extensions (Picket Fence)

  • Multi-task objectives: mutating across supervised, self-supervised, and reinforcement learning objectives simultaneously
  • Curriculum-aware switching: objective mutation conditioned on training progress (e.g., NTP early → JEPA late)
  • Federated SEVO: distributed evolution across multiple clusters with gossip-based population sharing
  • Meta-learning quotas: learning optimal diversity quotas from historical evolution runs rather than fixed percentages
  • Adaptive rungs: dynamically adjusting ASHA rung positions based on population convergence rate
  • Non-ternary extension: applying SEVO to float16/bfloat16 models with standard architectures
  • GPU cluster version: scaling SEVO to GPU farms with weight-copying via NVLink/InfiniBand
  • Reinforcement-learning objective selection: using RL to select which objective to assign instead of quota-based injection
  • Closed-loop integration with Arena for quality-aware evolution: using ELO-based model evaluation scores from an arena system as an additional fitness signal in SEVO population evolution, beyond perplexity
