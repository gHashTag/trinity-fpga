Problem Statement

Existing ternary neural networks treat dimension selection as a free hyperparameter, following Kaplan/Chinchilla scaling laws that predict monotonic improvement with increasing model capacity. No prior work identifies algebraic constraints linking ternary weight arithmetic to specific dimension families. FPGA inference of ternary transformers relies on DSP48 multiplier blocks (e.g., TerEffic 2025: 3,041 DSP on Alveo U280, $5,000+), limiting deployment to expensive accelerator cards.

Technical Disclosure

D001: Ternary Resonance Law (3k Dimensions)

All dimensions of a ternary {-1, 0, +1} neural network must be powers of 3 for optimal performance. Violating this constraint causes up to 2× perplexity degradation, even when increasing model capacity. The complete architecture uses exclusively 3k dimensions:

D002: Square Attention Theorem (ctx = head_dim)

When context length equals attention head dimension, the Q·KT attention matrix is square and can achieve full rank, maximizing information capacity. When ctx > head_dim, Q·KT is rank-deficient and positions collapse. In the HSLM configuration: head_dim = 81, ctx = 81 → square 81×81 matrix.

Sacred Attention Scale: 1/(d(φ−3)) = 0.354, versus the standard Vaswani (2017) scale 1/√d = 1/√81 ≈ 0.111. The sacred scale is 3.2× larger and stabilizes attention weights in the low-rank ternary space. Here φ = (1+√5)/2 is the golden ratio.

D003: Zero-DSP FPGA Ternary Inference

Complete transformer inference on FPGA using zero DSP48 multiplier blocks. Ternary MAC: each weight w ∈ {-1, 0, +1} encoded as 2 bits — w=+1 (01): sign-extend and add; w=−1 (11): negate via two's complement and add; w=0 (00): zero output. Cost: ~3 LUT per weight, zero multiplications. Shift-based RMSNorm via priority encoder + barrel shifter (~40 LUT, no DSP division). BRAM power-of-2 depth padding: 243×729 → 218 entries, out-of-range addresses return zero. Runtime UART weight loading with checksum validation. LFSR-driven hardware self-test with LED pass/fail.

Obvious Extensions (Picket Fence)

The following variations are disclosed to prevent trivial patent claims by third parties:

  • Extending context to 35=243 or 36=729 tokens while maintaining head_dim=ctx constraint
  • Expanding vocabulary to 37=2187 or 38=6561 tokens
  • Multi-FPGA pipeline: partitioning transformer blocks across 2–8 FPGAs connected via SPI/LVDS
  • 5-state quinary weights {-2,-1,0,+1,+2} with 3-bit encoding on the same LUT-only architecture
  • Mixed 3k/2n dimensions where only attention-critical dimensions are constrained to 3k
  • Applying resonance constraint to other balanced number bases (base-5, base-7)
  • Using φ-derived frequencies for other positional encodings (ALiBi, xPos)
  • Streaming inference with sliding-window attention at 3k window sizes
  • Pruning-aware training that maintains 3k structure after weight removal
