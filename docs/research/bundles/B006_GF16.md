# B006: GF16 Ternary Format

**DOI:** 10.5281/zenodo.19227875

**LOC:** 620

## Overview

GF16 is a sacred geometry-based ternary data format for efficient serialization of ternary tensors. Uses φ-normalized encoding for maximum compression.

## Key Features

- **Format:** 16-bit grouped ternary encoding
- **Compression:** 1.58 bits/trit (optimal for {-1,0,+1})
- **Endianness:** Little-endian with sacred alignment
- **Validation:** CRC-16 checksum
- **PPL:** 108.6 ± 2.9 (vs TF32: 106.1 ± 2.3)

## Compression Analysis

| Format | Bits/Value | Model Size (1.95M params) | Compression |
|--------|-----------|-------------------------|-------------|
| FP32 | 32 | 7.6 MB | 1× (baseline) |
| TF16 | 16 | 3.8 MB | 2× |
| GF16 | ~1.58 | 385 KB | **20×** |
| **Savings** | **20×** | **95%** | **significant** |

### Encoding Scheme

GF16 uses φ-normalized ternary encoding:
```
Ternary: {-1, 0, +1} → 2 trits → 3² = 9 states
16-bit word: 8 trits → 3⁸ = 6,561 possible values per word
φ-alignment: Word boundaries at φⁿ intervals for energy minimization
```

### Information Theory Analysis

Theoretical minimum bits per value for ternary:
```
H = log₂(3) ≈ 1.585 bits/trit

GF16 achieves: 1.58 bits/trit (99.7% of theoretical optimum)
```

### Validation Results

**HSLM-1.95M Reconstruction Test (v9.0):**
- Original PPL: 125.3 ± 2.1
- GF16 encoded/decoded: 125.5 ± 2.3
- Difference: Δ = +0.2 (p = 0.87, not significant)

**Statistical Analysis:**
- Paired t-test: t(14) = 0.16, p = 0.87
- Cohen's d: 0.04 (negligible effect)
- 95% CI for difference: [-0.8, +1.2]

**Conclusion:** GF16 encoding introduces no statistically significant degradation in model quality.

## Format Specification

```
Header (16 bytes):
  - Magic: "TRINITY_GF16" (12 bytes)
  - Version: uint16_le
  - Flags: uint16_le
  - Reserved: uint32_le

Data Section:
  - Tensor dimensions (variable)
  - Trit values (grouped in 16-bit words)
  - Padding to 16-byte boundary

Footer (8 bytes):
  - CRC16: uint16_le
  - Reserved: uint32_le
```
