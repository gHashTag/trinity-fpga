## Root Cause: Microglia Bug

**Bug location**: `evolution_simulation.zig:514-523`

```zig
fn microgliaPatrol(self: *EvolutionSimulator) u32 {
    var pruned: u32 = 0;
    for (self.workers[0..self.worker_count]) |*worker| {
        if (!worker.alive or worker.ppl > 100.0) {  // ← BUG
            worker.alive = false;
            pruned += 1;
        }
    }
    return pruned;
}
```

**Problem**:
- All workers start with PPL=500+ (line 38: `PplModel.A = 500.0`)
- Microglia patrol runs every 30 steps
- **Culls ALL workers with PPL > 100**, regardless of training stage
- This is **absolute threshold**, not relative

**Death spiral**:
1. Step 0-30: All workers PPL > 100
2. Step 30: First Microglia patrol → culls most workers
3. Step 31-46: Crash rate (10%) kills remaining
4. Step 47: Last worker dies → PPL = inf (division by zero)

## Byzantine Detection Logic

**Current behavior**:
- Byzantine rate: 5% (config)
- Detection rate: 15% per step (hardcoded)
- Expected: 100 × 5% × 15% = 0.75 detections/step
- Over 300 steps: 225 detections

**Actual results**:
- S4: 20 detections (workers died early from Microglia)
- S5: 10 detections (workers lived longer)

**Issue**: Byzantine detection happens **every step**, but dead workers don't generate events. S4's Microglia **competes** with Byzantine detection, killing workers before they can be detected.

## Recommendations

### 1. Fix Microglia (Priority HIGH)

Use **relative threshold** instead of absolute:

```zig
// Calculate median PPL of alive workers
// Cull only if PPL > 3× median (true outliers)
```

### 2. Separate Byzantine from Microglia (Priority HIGH)

- Byzantine: **Suspicion score** (increment on false report)
- Microglia: **Audit** (cull only if suspicion ≥ 3)

### 3. Fix S4/S5 Comparison (Priority HIGH)

- Same objectives (S4 has 3, S5 has 2)
- Same steps (S4 has 3×, S5 has 3×)
- Control only `microglia_interval` variable
